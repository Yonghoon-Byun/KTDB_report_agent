## 3. 🔴 확정된 문제

### 문제 A — 사회경제지표 시트: 3번째 컬럼 헤더 누락 (데이터팀 작업)

**현상**

사회경제지표 스프레드시트의 다음 6개 탭 전부에서, **C1 셀(3번째 컬럼 헤더)이 빈 칸**입니다.

| 탭 | 현재 헤더 (1행) |
|----|----------------|
| `POP_TOT` | `"SIDO", "SIGU", "", "2023", "2025", ...` |
| `POP_YNG` | `"SIDO", "SIGU", "", "2023", "2025", ...` |
| `POP_15P` | `"SIDO", "SIGU", "", "2023", "2025", ...` |
| `EMP` | `"SIDO", "SIGU", "", "2023", "2025", ...` |
| `STU` | `"SIDO", "SIGU", "", "2023", "2025", ...` |
| `WORK_TOT` | `"SIDO", "SIGU", "", "2023", "2025", ...` |

반면 같은 스프레드시트의 `ZONE` 탭은 정상입니다:

```
"SIDO", "SIGU", "ZONE"   ← 정상
```

**실제 값은 들어있음**

3번째 컬럼의 **데이터 자체**는 존번호가 들어있습니다:
```
"서울특별시", "종로구", "1", "141860", ...
                    ↑
              값은 정상 (ZONE 번호 1)
```

**문제가 되는 지점**

`streamlit_app.py:124` 코드:
```python
df = df.loc[:, df.columns != ""]
```
→ 이 코드가 **빈 헤더 컬럼을 자동 제거**합니다. 결과적으로 6개 탭에서 **ZONE 번호 컬럼이 통째로 사라집니다.**

**부가 이슈**: `POP_TOT` 탭은 2050년 이후로 빈 컬럼 7개 더 있음:
```
"SIDO", "SIGU", "", "2023", ..., "2050", "", "", "", "", "", "", ""
```

**수정 방법 (데이터 담당자, 약 2분)**

Google Sheets에서 각 탭을 열고 **C1 셀에 `ZONE` 입력**:

| 수정 전 | 수정 후 |
|---------|---------|
| `SIDO \| SIGU \| (빈칸) \| 2023 \| ...` | `SIDO \| SIGU \| ZONE \| 2023 \| ...` |

6개 탭 전부 동일하게 수정. POP_TOT의 오른쪽 빈 컬럼 7개는 선택 사항(삭제해도 좋음, 앱이 자동 무시하므로 그냥 둬도 동작은 함).

### 문제 B — 275행 코드 버그: 콤마 포함 숫자 처리 실패 (개발자 작업)

**사용자가 본 에러 메시지가 이것 때문에 발생합니다.**

```
❌ 데이터 로드 실패: invalid error value specified
```

**원인 분석**

OD 데이터(목적OD, 주수단OD, 접근수단OD)는 숫자에 **콤마**가 포함된 문자열로 저장되어 있습니다:
```
"ORGN", "DEST", "WORK", "SCHO", "BUSI", ...
"1",    "1",    "19,800", "2,839", "10,588", ...
                  ↑
              문자열 "19,800" (콤마 포함)
```

`streamlit_app.py:275` 현재 코드:
```python
df[col] = pd.to_numeric(df[col], errors="ignore")
```

이 코드는:
1. `"19,800"` 같은 문자열을 숫자로 변환 시도 → 실패 (콤마 때문)
2. `errors="ignore"` 옵션으로 원본 유지하려 함
3. **하지만 pandas 2.2부터 `errors="ignore"` 옵션이 제거됨** → `ValueError: invalid error value specified` 발생

CLAUDE.md에도 이미 **금지 규칙**으로 명시되어 있었는데(`"ignore" 사용 금지, "coerce" 사용`), 코드 반영이 안 되어 있었습니다.

**수정 방법 (개발자, 약 5분)**

`streamlit_app.py` 271-285행의 `preprocess()` 함수에서 275행 교체:

**수정 전**
```python
def preprocess(df: pd.DataFrame) -> pd.DataFrame:
    df = df.copy()
    for col in df.columns:
        if col not in ("SIDO", "SIGU"):
            df[col] = pd.to_numeric(df[col], errors="ignore")  # ← 버그
    ...
```

**수정 후**
```python
def preprocess(df: pd.DataFrame) -> pd.DataFrame:
    df = df.copy()
    for col in df.columns:
        if col not in ("SIDO", "SIGU"):
            # 콤마 제거 후 숫자 변환 시도
            cleaned = df[col].astype(str).str.replace(",", "", regex=False)
            converted = pd.to_numeric(cleaned, errors="coerce")
            if converted.notna().any():
                df[col] = converted
            # 전부 NaN이면 원본 유지 (문자열 컬럼 보호)
    ...
```

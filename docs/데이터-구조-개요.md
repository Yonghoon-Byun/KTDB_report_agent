# KTDB 데이터 구조 개요 (비개발자용)

> 이 문서는 KTDB 분석 에이전트가 어떤 데이터를 들고 있는지 한눈에 보여주는 지도입니다.
> "이런 질문을 해도 될까?"를 판단할 때 참고하세요. SQL은 몰라도 됩니다.

---

## 1. 한눈에 보기

```
                       ┌──────────────────────┐
                       │   수도권 1,310개 존    │  ← 위치 단위
                       │   (1,137 권역내부 +   │
                       │    173 권역외부)      │
                       └──────────┬───────────┘
                                  │
       ┌──────────────────────────┼────────────────────────────┐
       │                          │                            │
       ▼                          ▼                            ▼
 [사회경제지표]               [통행 OD]                  [참조 정보]
 사람 수 데이터               누가 어디로 갔나           SIDO/시군구
 (인구·취업·학생 등)          (5목적 × 10수단)         행정구역 매핑
```

데이터는 크게 **3가지 카테고리**, 7개 연도(`2023, 2025, 2030, 2035, 2040, 2045, 2050`)로 구성됩니다.

---

## 2. 지역 단위 — 1,310개 존

KTDB 수도권 분석의 최소 단위는 **TAZ (Traffic Analysis Zone, 교통 분석 존)** 입니다.

| 구분 | 존 개수 | 의미 |
|---|---|---|
| **권역내부** | 1,137개 | 서울/인천/경기 — 시군구·동 단위로 세분화 |
| **권역외부** | 173개 | 나머지 14개 시도 — 시도당 몇 개씩만 (시군구 단위 분석 ❌) |
| **합계** | **1,310개** | |

**예시**: "수원시"는 권선구/영통구/장안구/팔달구 4개 존, "부산광역시"는 1개 존만 존재.

📌 **분석 가능 범위**:
- ✅ 권역내부(서울/인천/경기) — 시군구 단위 자유롭게 분석
- ⚠️ 권역외부(다른 14개 시도) — 시도 합계까지만 의미, 시군구 단위는 데이터가 너무 듬성듬성

---

## 3. 카테고리 ① — 사회경제지표 (사람 수)

특정 지역에 사는/일하는/공부하는 **사람 수**를 연도별로 집계한 데이터.

| 데이터 종류 | 무엇을 알 수 있나 | 세분화 |
|---|---|---|
| **인구수** | 거주 인구 | 성별(남/여) × 연령(0~100세) |
| **취업자수** | 직장 있는 사람 수 | 성별 × 연령 |
| **종사자수** | 사업체에서 일하는 사람 수 | 총 종사자 / 3차산업 종사자 |
| **학생수** | 재학 중인 학생 수 | 초/중/고/특수/대학 |

**예상 가능한 질문**:
- "2025년 종로구 인구수"
- "강남구 65세 이상 인구 변화 (2023~2050)"
- "수도권 시군구별 대학생 분포 2030년"
- "성남시 3차산업 종사자 시계열"

---

## 4. 카테고리 ② — 통행 OD (사람의 이동)

**O**rigin → **D**estination, "출발지에서 도착지로 몇 명이 이동했는가" 데이터.
이게 KTDB의 핵심이고 데이터의 95%를 차지합니다 (5천만 행).

3가지 관점으로 쪼개져 있습니다:

### 4.1 목적OD — "왜" 갔나

| 목적 코드 | 의미 |
|---|---|
| `HOME` (귀가) | 다른 곳에서 집으로 |
| `WORK` (출근) | 집/타지에서 직장으로 |
| `SCHO` (등교) | 학교로 |
| `BUSI` (업무) | 업무차 외근/출장 |
| `OTHE` (기타) | 쇼핑, 친목, 기타 |

**예**: "2025년 종로구→강남구 출근 통행"

### 4.2 주수단OD — "무엇으로" 갔나

| 수단 코드 | 의미 |
|---|---|
| `walk_bike` | 도보·자전거 |
| `auto` | 승용차 |
| `taxi` | 택시 |
| `bus` | 버스 (광역·시내 통합) |
| `etc_bus` | 기타 버스 |
| `subw` | 지하철 |
| `bus_subw` | 버스+지하철 환승 |
| `rail` | 일반철도 (무궁화·새마을·ITX) |
| `ktx` | KTX |
| `freight` | 화물·기타 |

**예**: "서울→부산 KTX 통행 2025년"

### 4.3 목적별 주수단OD — "왜+무엇으로"

위 둘을 교차한 데이터. **4목적 × 10수단** (BUSI 제외).

**예**: "강남구 출근 + 승용차 OD 2030년" → 출근 목적이면서 승용차를 쓴 통행만

📌 **주의**: 목적별 주수단OD에는 **BUSI(업무) 없음** (원본 KTDB가 제공 안 함).

## 5. 데이터 규모 한눈에

| 카테고리 | 행수 (7년 합계) | 비유 |
|---|---|---|
| 존 마스터 (smr_zones) | 1,310 | 사전 |
| 사회경제지표 4종 | ~3.2M | 인구·취업·학생 통계 |
| 목적OD | 10.9M | "어디서 어디로 왜" |
| 주수단OD | 10.9M | "어디서 어디로 무엇으로" |
| 목적별 주수단OD | 28.8M | "어디서 어디로 왜+무엇으로" |
| **합계** | **~50.5M 행** | **약 7GB** |

## 6. 시간 축 — 7개 연도

| 연도 | 종류 | 의미 |
|---|---|---|
| 2023 | 기준연도 | 실제 관측·추정 |
| 2025 | 단기 | 5년 단위 장래 추정 |
| 2030 | 중기 | |
| 2035 | 중기 | |
| 2040 | 중장기 | |
| 2045 | 장기 | |
| 2050 | 최종 목표 | |

**중간 연도 (예: 2027, 2031)**: 데이터 없음 → 앱이 선형 보간 후 `*(보간)` 표시.

## 7. 자주 묻는 질문 패턴

자연어 채팅에 다음 키워드를 섞으면 앱이 자동으로 라우팅합니다:

| 사용자가 원하는 것 | 채팅에 포함할 키워드 |
|---|---|
| 특정 지역 한정 | 시도/시군구 이름 ("종로구", "수원시", "서울") |
| 특정 연도 | "2030년", "2025~2050", "장래", "기준연도" |
| 5년 단위 | "5년단위" |
| 시군구별 집계 | "시군구별", "시도별" |
| 도착지 분석 | "도착지 시군구별", "TOP 20" |
| 양단 (A↔B) | "서울 → 부산", "강남구에서 송파구로" |
| 인구 세분화 | "65세 이상", "여성", "초등학생", "3차산업" |
| 비율·증감률 | "분담률", "증감률", "CAGR", "연평균" |
| 단위 변환 | "천통행/일", "백만" |
| LLM 해설 추가 | "한 문장으로", "보고서 형식", "분석 요약" |
| 권역 필터 | "수도권" (권역내부만), "권역외부" |

## 8. 못 하는 분석 (데이터 한계)

| 질의 | 가능? | 이유 |
|---|---|---|
| 권역외부 시군구 단위 (예: 부산 해운대구) | ❌ | 권역외부는 시도당 몇 개 존만 보유 |
| 업무 목적 + 수단 교차 | ❌ | 목적별 주수단OD에 BUSI 없음 (원본 미제공) |
| 2027년처럼 보간연도 정확값 | ⚠️ | 선형 보간 추정치만 (`*(보간)` 표시) |
| 일별·월별·시간대별 통행 | ❌ | 연 단위 데이터만 |
| 개인 단위 OD | ❌ | 모두 존×존 집계 통계 |
| 화물 통행 OD | ⚠️ | 수단 `freight` 컬럼만, 목적/품목 분해 불가 |

## 🔴 이슈 B — 코드 버그 (긴급, 단일 병목)

### 증상
- 어떤 쿼리든 `❌ 데이터 로드 실패: invalid error value specified` 빨간 에러
- 사회경제지표·OD **모두** 영향

### 원인
- `streamlit_app.py:275`
  ```python
  df[col] = pd.to_numeric(df[col], errors="ignore")
  ```
- pandas 2.2부터 `errors="ignore"` 옵션이 제거됨 → 변환 성공 여부와 무관하게 `ValueError` 발생
- 추가로 OD 데이터는 숫자에 콤마 포함(`"19,800"`)이라, 옵션 수정만으로도 콤마 처리 누락

### 수정 방법
`streamlit_app.py` 271-285행 `preprocess` 함수에서 275행을 다음과 같이 변경:

```python
# 수정 전
df[col] = pd.to_numeric(df[col], errors="ignore")

# 수정 후
cleaned = df[col].astype(str).str.replace(",", "", regex=False)
converted = pd.to_numeric(cleaned, errors="coerce")
if converted.notna().any():
    df[col] = converted
# 전부 NaN이면 원본 유지 (문자열 컬럼 보호)
```

### 검증
1. Streamlit 재시작 (`streamlit run streamlit_app.py`)
2. 사이드바: 시도="경기도", 시군구="전체"
3. 입력: `경기도 2030년 인구수`
4. 빨간 에러 없이 표가 출력되면 ✅

### 메타
- **담당**: 개발자
- **소요**: 약 5분
- **우선순위**: 1순위 (이것 하나가 모든 쿼리를 막고 있음)

## 🟡 이슈 D — 의존성 누락 (배포 안전)

### 증상
- 로컬은 동작하지만 Streamlit Cloud 새 배포 시 `ImportError: No module named 'gspread'` 발생 가능

### 원인
- `requirements.txt` 현재 4줄:
  ```
  streamlit
  google-generativeai>=0.8.0
  st-gsheets-connection
  pandas
  ```
- `gspread`/`google-auth`가 `st-gsheets-connection`의 전이 의존성으로만 설치되고 있어, `st-gsheets-connection` 제거 시 즉시 깨짐

### 수정 방법
`requirements.txt`를 다음과 같이 **한 커밋에 동시 적용**:

```
streamlit
google-generativeai>=0.8.0
gspread>=5.0.0
google-auth>=2.0.0
pandas
```

- `st-gsheets-connection` 제거
- `gspread>=5.0.0` 추가
- `google-auth>=2.0.0` 추가

⚠️ **분리 커밋 금지**: `st-gsheets-connection`을 먼저 제거하면 `gspread`가 전이 의존성에서 빠지면서 `ImportError`가 발생합니다. 반드시 한 번의 편집으로 동시에 적용해주세요.

### 검증
- 로컬: `pip install -r requirements.txt` 실행 시 에러 없음
- Streamlit Cloud: `git push` 후 자동 재배포 로그에 `ImportError` 없음

### 메타
- **담당**: 개발자
- **소요**: 약 3분

## 🟡 이슈 C — OD 지역 필터링 (기능 보강)

### 증상
- 사이드바에서 "경기도" 선택 후 OD 쿼리(예: `2023년 출근 통행량`) 입력 시 **필터가 작동 안 함**, 전국 데이터가 그대로 출력됨

### 원인
- OD 데이터(목적OD, 주수단OD)에는 `ORGN`(출발존번호)만 있고 `SIDO`/`SIGU` 컬럼이 없음
- `streamlit_app.py:276-279`의 SIDO/SIGU 필터링이 OD 데이터에서는 적용 불가
- 코드에 ZONE 탭(`ORGN ↔ ZONE` 매핑)을 조인하는 로직이 없음

### 수정 방법
`streamlit_app.py`의 `preprocess` 함수(271-285행)에 ZONE 조인 로직 추가:

**요구사항**
1. `ORGN` 컬럼이 있을 때만 동작 (사회경제지표는 건너뛰기)
2. ZONE 탭(`load_sheet(SHEET_URL_SOCIO, "ZONE")`)을 로드해서 `ORGN = ZONE` 기준으로 `SIDO`/`SIGU` 컬럼 머지
3. **반드시 `try/except`로 감싸고**, 실패 시 `st.warning("지역 필터링 불가: ZONE 데이터 로드 실패")` 표시 후 SIDO/SIGU 없이 진행 (현행 동작 유지 = 열화 폴백)
4. 머지 후 기존 SIDO/SIGU 필터링 로직(`streamlit_app.py:276-279`)이 정상 동작하는지 확인

**참고**
- 진입점: `load_integrated()` 함수 (329행 부근)가 `preprocess()`를 호출
- 기존 ZONE 로드 예시: `get_sigu_list()` 함수 (141행)가 `load_sheet(url, "ZONE")` 사용
- 상세 설계: `.omc/plans/diagnose-and-fix-sheets-connectivity.md` Option 2 참조

### 검증
1. Streamlit 재시작
2. 사이드바: 시도="서울특별시", 시군구="전체"
3. 입력: `2023년 출근 통행량`
4. 표에 서울특별시 관련 존(`ORGN`)만 출력되면 ✅
5. 만약 ZONE 탭 로드가 실패해 노란 `st.warning`이 뜨면 부분 성공 (필터링은 안 되지만 크래시 없음)

### 메타
- **담당**: 개발자
- **소요**: 약 30분
- **우선순위**: 3순위 (B/D 완료 후)
